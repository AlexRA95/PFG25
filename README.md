## **Sobre el Proyecto: Abatech**

**Abatech** es más que un comercio electrónico; es una propuesta de transformación digital diseñada para ayudar a las empresas a adaptarse a las necesidades de un mercado cada vez más competitivo y digitalizado. El proyecto tiene como objetivo ofrecer una plataforma accesible, eficiente y escalable que no solo simplifique la experiencia de compra para los usuarios, sino que también permita a las empresas optimizar sus procesos y mejorar su competitividad.

Este desarrollo se estructura en **cinco fases clave**, que han sido diseñadas cuidadosamente para asegurar una implementación sólida y eficaz. Cada una de estas fases representa un paso crítico hacia la realización del proyecto:

---

### **Fases del Proyecto**

1. **Análisis de la Empresa**  
   En esta primera etapa, se lleva a cabo un diagnóstico exhaustivo para identificar las necesidades específicas del negocio. Este análisis permite evaluar los procesos actuales, identificar las áreas de mejora y definir un plan estratégico que sirva como base para las siguientes fases.

2. **Digitalización de los Procesos**  
   Una vez identificadas las áreas clave, se procede a transformar los procesos empresariales mediante el uso de tecnologías modernas. El objetivo principal es optimizar la eficiencia y reducir costos al mismo tiempo que se adapta el negocio a un entorno digital.

3. **Implementación de los Procesos**  
   Esta fase consiste en integrar las herramientas digitales diseñadas en los flujos de trabajo del día a día de la empresa. Se configuran sistemas, se establecen procesos y se garantiza que todo esté alineado con los objetivos estratégicos del negocio.

4. **Formación**  
   Reconociendo que las personas son el eje de cualquier transformación exitosa, esta etapa se enfoca en la capacitación del equipo humano. La formación permite al personal familiarizarse con las herramientas y procesos digitales, asegurando que puedan utilizarlos de manera eficiente.

5. **Acompañamiento**  
   Finalmente, esta fase proporciona un soporte continuo y una supervisión personalizada para garantizar que los cambios implementados sean adoptados correctamente. También incluye el seguimiento de resultados, ajustes necesarios y la evaluación del impacto.

# **Cronograma del Proyecto Abatech**

El desarrollo del proyecto **Abatech** se divide en cinco fases principales, distribuidas en un cronograma que abarca un período de **5 meses**. Cada fase tiene objetivos específicos y un plazo estimado para su realización.

---

## **Cronograma General**

| **Fase**                     | **Descripción**                                   | **Duración**    | **Periodo Estimado**     |
|-------------------------------|--------------------------------------------------|-----------------|--------------------------|
| 1. Análisis de la Empresa     | Estudio inicial de la empresa y recopilación de datos. | 3 semanas       | Febrero 1 - Febrero 21   |

## **Detalles por Fase**

### **Fase 1: Análisis de la Empresa** (3 semanas)
- **Objetivo**: Identificar las necesidades específicas de la empresa y definir los requisitos técnicos.
- **Actividades**:
   - Reuniones iniciales con los responsables de la empresa.
   - Análisis de los procesos actuales y áreas de mejora.
   - Creación del documento de especificaciones funcionales.
- **Resultados esperados**: Informe detallado sobre los requisitos del proyecto.

---

### **Fase 2: Digitalización de Procesos** (4 semanas)
- **Objetivo**: Diseñar y estructurar los procesos digitales según las necesidades detectadas.
- **Actividades**:
   - Creación del diseño técnico (arquitectura del sistema).
   - Selección de las herramientas y tecnologías a utilizar.
   - Desarrollo de prototipos funcionales.
- **Resultados esperados**: Prototipo inicial y documentación técnica.

---

### **Fase 3: Implementación** (6 semanas)
- **Objetivo**: Desarrollar, probar e implementar las herramientas y sistemas digitales.
- **Actividades**:
   - Desarrollo del backend y frontend del sistema.
   - Configuración de la base de datos.
   - Pruebas unitarias e integración de los módulos desarrollados.
- **Resultados esperados**: Sistema e-commerce completamente funcional.

---

### **Fase 4: Formación** (2 semanas)
- **Objetivo**: Capacitar al personal de la empresa en el uso de las herramientas digitales.
- **Actividades**:
   - Elaboración de manuales y guías de usuario.
   - Realización de sesiones formativas para el personal.
   - Resolución de dudas y evaluación de competencias.
- **Resultados esperados**: Personal capacitado para operar el sistema.

---

### **Fase 5: Acompañamiento** (3 semanas)
- **Objetivo**: Ofrecer soporte técnico y garantizar la correcta adopción del sistema.
- **Actividades**:
   - Resolución de problemas técnicos.
   - Monitoreo de resultados e impacto del sistema.
   - Realización de ajustes y optimizaciones.
- **Resultados esperados**: Sistema optimizado y en pleno funcionamiento.

## **Tecnologías Utilizadas**

El desarrollo del proyecto **Abatech** se llevó a cabo utilizando un conjunto de tecnologías modernas y probadas que garantizan un diseño robusto, eficiente y escalable. A continuación, se detallan las herramientas y frameworks empleados:

### **Lenguajes y Entorno**
- **Java 11**: El núcleo del proyecto, proporcionando una base sólida y moderna para la lógica del negocio.
- **Java EE 7**: Plataforma empresarial para desarrollar aplicaciones web escalables y seguras.

### **Frameworks y Arquitectura**
- **Modelo Vista Controlador (MVC)**: Arquitectura de software que separa la lógica del negocio, la vista y el controlador, mejorando la modularidad y facilitando el mantenimiento.

### **Frontend**
- **HTML5**: Estructura base para la creación de interfaces web dinámicas y semánticas.
- **CSS3**: Diseño estilizado para garantizar una experiencia visual atractiva y profesional.
- **Bootstrap**: Framework de CSS utilizado para crear diseños responsivos y optimizados para dispositivos móviles.
- **JSP (JavaServer Pages)**: Tecnología para generar contenido dinámico en el lado del servidor, utilizada en la capa de presentación para interactuar con los datos del modelo.

### **Gestión de Dependencias**
- **Maven**: Herramienta de gestión de proyectos utilizada para manejar dependencias, compilar el código y construir el proyecto.

---

## **Arquitectura del Proyecto**

El proyecto sigue el patrón **Modelo-Vista-Controlador (MVC)** para separar las responsabilidades y mejorar la escalabilidad del sistema. La estructura se organiza de la siguiente forma:

1. **Modelo**:
   - Representa la lógica del negocio.
   - Gestiona los datos de la aplicación, incluyendo las interacciones con la base de datos.

2. **Vista**:
   - Implementada con **JSP** para generar contenido dinámico.
   - Diseñada utilizando **HTML5**, **CSS3** y **Bootstrap** para garantizar una experiencia de usuario moderna y responsiva.

3. **Controlador**:
   - Gestiona la comunicación entre el modelo y la vista.
   - Procesa las solicitudes del usuario (HTTP) y delega las acciones al modelo o la vista según sea necesario.

## **Requisitos del Proyecto**

Para ejecutar el proyecto, se requiere lo siguiente:

- **JDK 11** o superior.
- **Maven** para la gestión de dependencias.
- Un servidor de aplicaciones compatible con **Java EE 7** (como GlassFish o WildFly).
- Un navegador moderno para visualizar la interfaz web.

## Instalación y Configuración
1. Clona el repositorio del proyecto:
   ```bash
   git clone https://github.com/AlexRA95/PFG25.git
   ```

2. Instala las dependencias necesarias para el backend:
   ```bash
   cd backend
   npm install
   ```

3. Configura las variables de entorno en un archivo .env para conectar con la base de datos y otros servicios.

4. Inicia el servidor backend:
   ```bash
   npm start
   ```

5. Instala las dependencias necesarias para el frontend:
   ```bash
   cd frontend
   npm install
   ```

6. Inicia el servidor de desarrollo para el frontend:
   ```bash
   npm start
   ```
